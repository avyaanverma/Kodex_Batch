# HTML Complete Notes (From Zero to Solid)

## 1. What is HTML?
HTML (HyperText Markup Language) is **not a programming language**.  
It is a **markup language** used to structure content on the web.

- Defines **structure**, not logic
- Tells the browser **what content is**, not **how it behaves**
- Works with:
  - CSS → styling
  - JavaScript → behavior

## 2. Basic HTML Document Structure

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    Content goes here
</body>
</html>
````

### Explanation

* `<!DOCTYPE html>` → HTML5 declaration
* `<html>` → root element
* `<head>` → metadata (not visible)
* `<body>` → visible content

## 12. Semantic HTML (VERY IMPORTANT)

❌ Bad:

```html
<div class="header"></div>
<div class="nav"></div>
```

✅ Good:

```html
<header></header>
<nav></nav>
<main></main>
<section></section>
<article></article>
<aside></aside>
<footer></footer>
```

### Why semantic HTML?

* Better SEO
* Better accessibility
* Cleaner structure
* Screen readers understand content

---

## 13. Block vs Inline Elements

### Block

* Takes full width
* Starts on new line

Examples:

```html
<div>, <p>, <h1>, <ul>, <section>
```

### Inline

* Takes required width
* Same line

Examples:

```html
<span>, <a>, <img>, <strong>
```

## 21. Accessibility Basics

* Always use `alt` in images
* Use semantic tags
* Label form inputs

```html

<input id="name">
```

## 22. HTML Best Practices (READ THIS)

* Use semantic HTML
* Avoid inline styles
* One `<h1>` only
* Proper indentation
* Use lowercase tags
* Close all tags properly

---

## 23. HTML Is NOT Responsible For

* Styling → CSS
* Logic → JavaScript
* Database → Backend

HTML = **Structure only**
